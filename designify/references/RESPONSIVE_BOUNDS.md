# Responsive Bounds

## Principle

A good desktop composition is not automatically a good website. The design must survive smaller screens without losing hierarchy or becoming visually dead.

## Rules

- preserve the main visual anchor on tablet and mobile
- remove or reduce decorative layers before they create clutter
- collapse complexity, not quality
- keep line lengths readable
- keep buttons and controls comfortable to tap
- avoid depending on precise overlap to make the design work
- reduce animation complexity on smaller devices where necessary

## Common mistakes

- shrinking everything instead of recomposing it
- stacking into a boring, endless vertical list
- leaving enormous empty gaps caused by desktop spacing values
- keeping decorative absolute layers that now block content
- preserving desktop hover behavior without adapting for touch

## Good mobile behavior

A premium mobile page should still feel:

- intentional
- paced
- readable
- tactile
- branded
- fast enough to feel real
