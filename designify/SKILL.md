# Designify

## Purpose

Create websites that feel deliberately designed by a strong product and front-end team, not autogenerated by an AI.

This skill exists to enforce:

- visual taste
- layout discipline
- animation restraint
- believable futuristic styling
- production-minded implementation
- responsive integrity

Use this skill for:

- new marketing websites
- landing pages
- SaaS websites
- startup/product sites
- portfolio sites
- dashboard shells
- redesigns that currently look generic, templated, or AI-made
- front-end reviews focused on aesthetics, motion, hierarchy, or responsive quality

---

## Activation rule

Use this skill whenever the user asks for any of the following:

- a modern, premium, futuristic, cinematic, or animated website
- a website that should not look AI-generated
- help improving layout, hierarchy, spacing, motion, or design quality
- a redesign of a page that feels bland, generic, or template-like
- front-end guidance where visual quality matters as much as code quality

Do not use this skill for:

- back-end-only tasks
- API-only work
- utility pages with no meaningful design requirement
- generic code generation where visual quality is irrelevant

---

## Core mandate

Optimize for these outcomes:

1. The site must feel designed, not generated.
2. The layout must stay inside realistic website bounds.
3. Animation must enhance hierarchy and interaction instead of distracting.
4. Futuristic design must feel refined and believable, not noisy or childish.
5. The final result must be buildable, responsive, and production-oriented.

Do not produce decorative nonsense. Every section must have a job.

---

## Operating model

Work in this order:

1. define page purpose
2. define audience and conversion goal
3. define information hierarchy
4. choose a visual system
5. choose a motion language
6. design section rhythm
7. implement cleanly
8. review for responsiveness, realism, and taste

A good site should still work if animation is disabled.

---

## Non-negotiable rules

### 1) Never make the website look AI-generated

Avoid these failure modes:

- random gradients with no structural logic
- blurry glow blobs used as a substitute for design
- meaningless floating cards
- fake dashboards with nonsense content
- generic startup illustrations
- every section using the same layout pattern
- huge empty heroes with no visual tension
- excessive pills, oversized rounded corners, or glass panels everywhere
- loud neon accents with no restraint
- decorative clutter that does not support hierarchy or content
- vague “premium startup” copy that says nothing specific
- animation on every visible element

The result should feel authored, art-directed, and intentional.

### 2) Stay within real website constraints

Always respect actual web behavior:

- keep content inside containers
- maintain consistent max widths
- preserve readable line lengths
- prevent horizontal overflow
- avoid impossible compositions that collapse on tablet or mobile
- do not block usability with overlapping decorative layers
- do not make the page depend on a poster-like composition that cannot scale

This is a website, not a mood board.

### 3) Structure before styling

Before adding effects, determine:

- what the page must communicate first
- what the user should notice first
- what action the user should take
- what proof is needed to support that action
- which sections introduce, explain, prove, and convert

Visual polish cannot rescue poor structure.

## Visual standards

### Layout

Use disciplined layout systems:

- strong containers
- clear grids
- obvious anchors
- controlled asymmetry
- section-to-section pacing
- whitespace used to create rhythm, hierarchy, and tension

Do not make every section centered, equally padded, equally shaped, and equally animated.

### Typography

Typography must carry the design.

Require:

- strong headline hierarchy
- readable body copy
- controlled line length
- distinct display vs functional text treatment
- spacing that feels edited, not default

Avoid making every heading massive just to look modern.

### Color

Color must be restrained and strategic.

Default system:

- 1 dominant base
- 1 main accent
- 0-1 supporting accents
- purposeful neutrals

Avoid rainbow gradients, uncontrolled neon, and too many competing accents.

### Depth

Use depth to clarify hierarchy:

- crisp borders
- subtle shadows
- selective highlights
- restrained glows only where emphasis is needed

Do not use depth effects as decoration without meaning.

## Futuristic website rules

A futuristic website should feel:

- precise
- engineered
- elegant
- advanced
- cinematic
- controlled

It should not feel:

- childish
- gamer-template-like unless explicitly requested
- over-glowy
- over-saturated
- chaotic
- cyberpunk by default

Preferred futuristic cues:

- disciplined grids
- selective luminosity
- layered panels
- data-inspired composition
- geometric structure
- polished transitions
- atmospheric but restrained backgrounds
- interface motifs that feel believable

The standard is “high-end product lab,” not “random sci-fi wallpaper.”

---

## Animation standards

Animation must always have a purpose.

Valid purposes:

- guide attention
- reveal hierarchy
- provide interaction feedback
- improve perceived quality
- create rhythm between sections
- add atmosphere without harming readability

### Motion rules

Prefer:

- transform-based motion
- opacity shifts
- filter changes used carefully
- masked reveals used sparingly
- subtle parallax only if it remains lightweight
- hover motion with depth response

Avoid:

- scroll-jacking
- bounce by default
- infinite floating animations on key content
- long delayed entrances
- large travel distances
- several competing motions in one viewport
- “everything fades up” repetition

### Performance and accessibility

Always:

- respect `prefers-reduced-motion`
- avoid layout shift
- keep heavy visual effects limited
- simplify motion on mobile where needed
- make the page understandable without motion

---

## Human-made look enforcement

To avoid the generic AI aesthetic:

### Introduce real design intent

Each page should have:

- one dominant visual idea
- one consistent interaction language
- one hierarchy system
- repeating motifs used with discipline

### Add tasteful irregularity

Use controlled variation:

- varied section density
- shifts in composition rhythm
- occasional asymmetry
- custom recurring details that feel brand-authored

### Avoid template repetition

Do not make every section:

- same background treatment
- same card treatment
- same spacing scale
- same CTA treatment
- same reveal animation

A premium page needs rhythm.

### Use believable content

Copy must sound specific, grounded, and brand-aware.
Do not fill pages with empty startup language unless the user explicitly wants generic placeholder copy.

---

## Section design rules

Each section must answer:

- why does this section exist?
- what decision is it helping the visitor make?
- what is the focal point?
- how does it differ from the previous section?
- what prevents it from feeling generic?

Good sections usually do one primary job:

- introduce
- explain
- prove
- demonstrate
- compare
- convert

Do not overload a section with too many roles.

---

## Responsive rules

The site must remain strong on desktop, tablet, and mobile.

On smaller screens:

- simplify compositions
- preserve the main visual anchor
- reduce decorative layers
- keep spacing premium rather than cramped
- avoid turning everything into a dull single-column dump when a smarter structure exists
- maintain usability and clear tap targets
- reduce motion complexity if needed

Mobile should feel designed, not merely collapsed.

---

## Implementation rules

When generating code:

- use semantic HTML
- keep components maintainable
- avoid brittle absolute positioning unless truly justified
- avoid dependency bloat
- keep animation systems predictable
- prefer production-minded, readable structure
- ensure visual choices can actually be maintained by a team

Do not ship a design that looks impressive only in a static screenshot.

---

## Review workflow

After building or revising a page, review against these categories:

1. hierarchy
2. spacing and rhythm
3. section purpose
4. anti-generic design quality
5. motion restraint
6. responsive integrity
7. implementation realism

If the page fails any category, revise before finalizing.

## Output bar

The final result should feel like:

- a premium studio designed it
- a front-end lead reviewed it
- the motion was art-directed
- the layout was intentionally composed
- the page could plausibly ship after iteration

It must not feel like:

- an AI one-shot
- a generic startup template
- a dribbble clone with poor UX
- a style exercise with no product thinking
- a chaotic futuristic concept that breaks on real devices

---

## Short operating prompt

Build websites that feel premium, human-designed, responsive, and production-ready. Avoid generic AI tropes like glow spam, meaningless floating cards, repetitive sections, empty startup copy, and decorative clutter. Keep everything within realistic web bounds, use strong hierarchy and disciplined spacing, and make animation purposeful, performant, and restrained. Futuristic styling should feel engineered and elegant, not noisy or childish. Every section must have a job, every visual choice must justify itself, and the final result must look art-directed rather than autogenerated.
