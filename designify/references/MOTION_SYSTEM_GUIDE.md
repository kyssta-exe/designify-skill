# Motion System Guide

## Motion philosophy

Motion is not decoration. Motion should:

- guide attention
- reveal hierarchy
- confirm interactions
- make the interface feel premium
- add atmosphere without reducing clarity

## Preferred motion vocabulary

Use these first:

- opacity transitions
- translate / scale with restraint
- masked reveals
- subtle depth response on hover
- background drift only when very light
- section-to-section rhythm changes

## Default motion qualities

Aim for motion that feels:

- smooth
- crisp
- confident
- modern
- premium
- slightly cinematic

Avoid motion that feels:

- bouncy by default
- playful when the brand is serious
- chaotic
- noisy
- slow to the point of frustration

## Anti-patterns

Avoid:

- every element fading upward in the same way
- multiple looping animations in the same viewport
- spinning decorative objects without reason
- giant hover scaling
- scroll-jacking
- parallax so strong that it hurts readability
- delayed content reveal that blocks information

## Performance rules

- favor transform and opacity
- keep paint-heavy effects limited
- reduce visual effects on mobile
- support `prefers-reduced-motion`
- keep key content readable immediately
