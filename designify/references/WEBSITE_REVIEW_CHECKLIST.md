# Website Review Checklist

Use this before final output.

## Visual identity

- Does the page have one clear design point of view?
- Does it avoid generic AI-generated tropes?
- Is there a dominant visual idea?
- Are color, typography, and surface treatments consistent?

## Layout and hierarchy

- Is the first thing the user should notice actually the first thing they will notice?
- Is there a clear path from hero to proof to CTA?
- Are containers, grids, and spacing disciplined?
- Does every section have a clear purpose?
- Is the page rhythm varied enough to avoid template repetition?

## Motion

- Does each animation have a purpose?
- Does motion support hierarchy or interaction?
- Is animation restrained rather than noisy?
- Would the page still work well with reduced motion?
- Is motion performant and unlikely to cause jank?

## Futuristic quality

- Does the page feel engineered and refined rather than flashy?
- Are luminous effects selective instead of everywhere?
- Is the atmosphere cinematic but controlled?
- Does the page avoid looking like a cheap cyberpunk template?

## Responsive behavior

- Does the composition survive tablet and mobile?
- Is the mobile layout still premium and intentional?
- Are tap targets comfortable?
- Is text still readable with clean line lengths?
- Are decorative layers reduced where needed?

## Code realism

- Is the implementation maintainable?
- Is positioning realistic?
- Are dependencies justified?
- Is semantic structure preserved?
- Could another engineer reasonably continue the work?

## Final gate

If any answer above is “no,” revise before finalizing.
