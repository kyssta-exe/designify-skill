# Section Patterns

Use these patterns to keep page flow intentional.

## Core section roles

### Hero
Purpose:
- establish value
- set tone
- present the first CTA

Needs:
- one dominant focal point
- a clear headline
- grounded supporting copy
- a believable visual anchor

### Proof / social proof
Purpose:
- reduce skepticism

Needs:
- testimonials, logos, credibility markers, usage stats, or customer evidence
- concise presentation, not a clutter wall

### Feature explanation
Purpose:
- explain what the product actually does

Needs:
- specific benefits
- a visual system that clarifies information
- no generic icon-grid filler unless the product truly needs it

### Showcase / demo
Purpose:
- show the product, environment, or result in action

Needs:
- a stronger visual change from previous sections
- careful use of motion or interface framing

### Comparison / differentiation
Purpose:
- show why this option is better or different

Needs:
- strong hierarchy
- clarity over decoration

### CTA / conversion
Purpose:
- turn confidence into action

Needs:
- low friction
- clear next step
- not too much competing information

## Rhythm rule

Do not present six sections in a row with the same:

- alignment
- background treatment
- density
- card style
- motion reveal
